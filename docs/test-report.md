# 테스트 보고서

## 1. 테스트 목적
구현한 화면이 실제로 실행되는지 확인하고, MVP 범위에서 주요 동작이 정상적으로 보이는지 점검한다. 특히 이번 테스트에서는 **로그인 보호 흐름**, **AI 상담 UX**, **이체 3단계 시뮬레이션**이 정상적으로 동작하는지 함께 확인한다.

## 2. 테스트 대상
- 메인 페이지
- 로그인 페이지
- 사용자 대시보드 페이지
- 금융상품 페이지
- AI 챗봇 페이지
- 이체 시뮬레이션 페이지
- 보호 페이지 접근 제어

## 3. 실행한 명령어
```bash
npm install
npm run lint
npm run build
```

추가로 시연 안정성을 위해 아래 실행 방식도 확인했다.

```bash
npm run start
```

## 4. 기능 테스트 항목

| 항목 | 확인 내용 | 기대 결과 |
| --- | --- | --- |
| 메인 페이지 | 주요 섹션 렌더링 | 서비스 개요와 CTA 노출 |
| 로그인 | 입력값 검증 및 데모 계정 로그인 | 성공 시 보호 페이지 접근 가능 |
| 보호 페이지 | 비로그인 상태 접근 시 로그인 유도 | `/login`으로 이동 |
| 대시보드 | 자산/소비/거래 렌더링 | 더미 데이터 정상 노출 |
| 금융상품 | 상품 카드 렌더링 | 상품 설명과 특징 표시 |
| AI 챗봇 | 질문 입력 후 응답 생성 | 상담 유형과 추천 근거 표시 |
| 이체 시뮬레이션 | 입력, 확인, 완료 흐름 | 단계별 검증 및 완료 정보 출력 |

## 5. 실행 결과

- `npm install`: 성공
- `npm run lint`: 성공, ESLint 경고/오류 없음
- `npm run build`: 성공, 빌드 완료
- `npm run start`: 실행 가능, 발표용 시연 환경으로 사용 가능

### 빌드 확인 페이지
- `/`
- `/login`
- `/dashboard`
- `/products`
- `/ai-chat`
- `/transfer`

## 6. 수동 확인 결과

### 로그인 및 보호 페이지
- 데모 계정 입력 시 로그인 가능
- 잘못된 계정 입력 시 오류 메시지 출력
- 로그인 후 보호 페이지 접근 가능
- 비로그인 상태에서 보호 페이지 접근 시 로그인 페이지로 이동

### AI 챗봇
- 빠른 질문 버튼 동작 확인
- 질문 입력 시 응답 생성 중 상태 표시 확인
- 상담 유형 라벨 표시 확인
- 추천 근거 문구 표시 확인

### 이체 시뮬레이션
- 최소 금액 검증 동작 확인
- 잔액 부족 검증 동작 확인
- 입력 → 확인 → 완료 단계 전환 확인
- 완료 후 거래 번호 및 완료 시각 표시 확인

## 7. 확인된 이슈 및 대응
- `next dev` 환경에서 간헐적인 서버 오류가 발생할 수 있어, 시연 시에는 `npm run start` 사용을 권장한다.
- 실제 금융 API 연동이 아니므로 데이터 정확성보다는 화면 흐름과 상호작용 검증에 초점을 둔다.

## 8. 비고
이 프로젝트는 실제 금융 연동이 아닌 데모 웹사이트이므로, 기능 안정성 평가는 화면 렌더링, 입력 처리, 보호 페이지 접근 제어, 상담 시나리오 표현, 이체 시뮬레이션 동작을 중심으로 수행했다.
