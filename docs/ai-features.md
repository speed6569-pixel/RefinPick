# AI 기능 설계

## 1. 핵심 AI 기능
BankFlow의 핵심 AI 기능은 **금융 상담 챗봇**이다. 사용자의 질문에 따라 소비 패턴 요약, 저축 상품 추천, 소상공인 대출 상담 흐름을 안내한다.

현재 데모에서는 단순히 텍스트를 보여주는 수준을 넘어서, 아래 요소까지 함께 표현한다.

- 질문 의도에 따른 상담 분기
- 상담 유형 라벨 표시
- 추천 근거 또는 분석 기준 표시
- 실제 Amazon Lex 연동을 가정한 구조 설명

## 2. 왜 이 기능을 선택했는가
1주 프로젝트에서는 AI 기능이 반드시 화면에서 직접 보이는 것이 중요하다. 챗봇은 다음 장점이 있다.

- 사용자가 즉시 체험 가능
- AWS Lex 활용 방안을 설명하기 쉬움
- 금융 서비스와 자연스럽게 연결됨
- 발표 시 짧은 시간 안에 시나리오를 보여주기 좋음
- 상담 결과를 상품 추천 및 대출 안내로 연결하기 쉬움

## 3. 현재 구현 방식
현재 웹사이트에서는 **모의 응답 기반의 상담 UX**로 구현되어 있다.

### 현재 동작 방식
1. 사용자가 질문을 입력하거나 빠른 질문 버튼을 선택
2. 질문 키워드에 따라 소비 분석 / 저축 추천 / 대출 상담 / 일반 안내로 분기
3. 응답 생성 중 상태 메시지 표시
4. 상담 유형 라벨과 추천 근거를 함께 출력

### 구현된 상담 시나리오
- 이번 달 소비 패턴 요약
- 저축 상품 추천
- 소상공인 대출 상담 흐름

이 방식은 실제 Amazon Lex API가 없어도 AI 기능이 화면에서 동작하는 흐름과 UX를 충분히 보여줄 수 있다는 장점이 있다.

## 4. 현재 데모의 강점
- AI 기능이 단순 문구가 아니라 실제 사용자 입력 흐름과 연결됨
- 답변마다 상담 목적이 명확하게 보임
- 추천 근거가 함께 표시되어 신뢰도가 높아짐
- 금융 서비스 화면 안에서 AI가 어떤 역할을 하는지 설명하기 쉬움

## 5. 실제 AWS Lex 연동 설계

### 사용 리소스
- Amazon Lex
- Amazon API Gateway
- AWS Lambda
- Amazon CloudWatch
- 필요 시 DynamoDB 또는 상품 데이터 저장소

### 예상 처리 흐름
1. 프론트엔드에서 사용자 질문 전송
2. API Gateway가 요청 수신
3. Lambda가 사용자 질문과 금융 컨텍스트를 조합해 Lex 요청 생성
4. Lex가 의도 분류와 슬롯 처리를 바탕으로 상담 응답 생성
5. Lambda가 응답을 요약/정제 후 프론트엔드에 반환
6. CloudWatch에 요청 로그 저장

## 6. 시연 가능한 질문 예시
- 이번 달 소비 패턴 요약해줘
- 저축 상품 추천해줘
- 소상공인 대출 상담 흐름 보여줘

## 7. 발표 시 강조 포인트
- AI 기능이 독립된 챗봇이 아니라 금융 서비스 흐름 안에 포함되어 있음
- 사용자 질문을 단순 답변이 아니라 소비 분석, 상품 추천, 대출 상담으로 연결함
- 실제 운영 시에는 Lex와 내부 데이터 조합으로 고도화 가능함
- 현재 데모는 UX 검증에 집중하고, 구조는 AWS 확장을 고려해 설계됨

## 8. 기대 효과
- 단순 정보 조회보다 더 높은 사용자 경험 제공
- 금융상품 안내를 개인화된 상담처럼 표현 가능
- AWS AI/ML 리소스 활용 능력을 평가 항목에 맞게 설명 가능
- 발표 시 AI 기능을 가장 직관적으로 시연 가능

## 9. 향후 확장
- 거래 데이터 기반 개인화 추천 고도화
- Lex 의도/슬롯 설계 고도화
- 상담 기록 저장 및 후속 액션 추천
- 상품 추천 근거에 실제 더미 데이터 또는 API 결과 연결
- 신용평가 보조 모델과 연계한 대출 상담 확장
